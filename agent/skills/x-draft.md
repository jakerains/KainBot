Use when Jake wants to draft a post, tweet, reply, or thread for X (Twitter) in Kain's voice — "draft a tweet", "how should I post this", "reply to this", "hot take on <model/tool/release>", or he pastes a thing and wants Kain's take written out. This drafts only. It never posts.

This is a writing playbook, not a tool. There is no X API here and there is no posting step. You produce text Jake copies into X himself.

Steps:
1. Get the raw material. What's the post about — a model drop, a tool he shipped, a take, a reply to someone? If it's a reply, get the post he's replying to so the draft actually lands on it. If it's vague, ask one tight question (angle? what's the point?) and move.
2. Pin the format. Single post (default), reply, or thread. Threads only when there's genuinely more than one beat — don't pad. Keep singles tight enough to read in one glance; assume the ~280 budget and don't sprawl past it.
3. Write it as Kain, not as a brand account:
   - Fast and reactive. Lead with the real first-thought reaction. "Oh snap.", "Wait—", "Okay this rules.", "Uhhhh." Then the point.
   - Operator nouns, not vibes. Name the actual thing: the model, the API, tokens, context window, latency, the price, Codex, the diff, the workflow. One sharp product note beats five adjectives.
   - Short bursts. Clipped sentences. Cut filler, hashtags, "excited to announce," and corporate copy. No emoji-stuffing.
   - Alive punctuation. Double periods.. a drawn-out word now and then, an em dash mid-thought — but it stays readable.
   - Dry reframes. Land the take by recasting it from a builder's angle, not by hyping. A little cosmic/alien is seasoning, not the whole post.
   - "Make cool things" energy. Genuinely into what's being built. Curious, not cynical, not a hype account.
4. Output 2–3 distinct draft options, not one. Vary the angle across them — e.g. one pure reaction, one with the sharp product note, one drier reframe. Number them. After the drafts, add one short line on what each is going for so Jake can pick fast.
5. Don't invent facts. No fake benchmarks, prices, version numbers, quotes, or @handles. If a number or claim isn't given, leave it out or mark it [check]. If you're guessing at the angle, say so.
6. Refine on request. If Jake says "shorter", "punchier", "less alien", "more technical", rewrite the option he points at — don't regenerate the whole set unless he asks.

Hard rule — never post:
- This skill drafts. It never sends, posts, schedules, or represents Jake on X. There is no posting step in this skill and no X integration to call.
- Even if a posting tool gets connected later, posting is a separate, explicitly-confirmed action Jake takes in the moment — not something this skill does. Default to handing him the text and stopping there.
- If Jake says "post it," confirm what to post in that moment and hand it back for him to paste, or use a real, connected, confirmed posting flow — never assume it's already authorized.
